## 1. Purpose & user journey
Villa-Returns P2 monthly operations dashboard: shows assigned villas as a year × month matrix with per-cell status (published / approved / submitted / draft / missing / rejected), an action queue banner for required submissions, 5-tile KPI strip, year tabs, status filter tabs (All / Needs action / Missing docs / Rejected), and a mobile card list. Each cell links to the per-period submit form.

## 2. Frontend structure
- Sidebar + mobile menu + shared `developer-topbar.html` (`operations-dashboard.html:4-14`).
- Skeleton loader (`#ops-skeleton`) revealed initially, hidden when data loads (`operations-dashboard.html:19-33`).
- 5-card stats grid (`#ops-stats`): missing / drafts / awaiting review / published / docs missing (`operations-dashboard.html:36-107`).
- "Nothing due yet" info banner for fresh villas (`operations-dashboard.html:109-127`).
- Action queue card (`#ops-urgent-banner`) with dismiss button + per-asset chip rows (`operations-dashboard.html:130-152`).
- Mobile card list container (`#ops-mobile-list`) — rendered only at narrow viewports (`operations-dashboard.html:155`).
- Matrix card with year tabs, filter tabs, sticky-column table, and legend (`operations-dashboard.html:158-217`).
- Empty state with hero illustration + 3 metric tiles + 3-step onboarding (`operations-dashboard.html:220-346`).
- Error block (`#ops-error`) with icon and refresh hint (`operations-dashboard.html:349-365`).
- No HTMX. All rendering done by `developer-operations-dashboard.js` after `DOMContentLoaded`.

## 3. Backend wiring
| Frontend call | Backend route | Handler | Status |
| --- | --- | --- | --- |
| `GET /api/developer/operations/dashboard?year=${year}` (`developer-operations-dashboard.js:44`) | `/api/developer/operations/dashboard` | `api_developer_operations_dashboard` — `backend/src/developer/villa_operations.rs:86` | wired, real DB |

Sub-bullets:
- Auth gate: `DeveloperUser` extractor (`backend/src/developer/extractors.rs:25-63`) — 401 if not authenticated, 403 if user lacks `developer`/`asset_owner`/`admin`/`super_admin` role.
- Data source: single CTE-rich SQL over `developer_asset_links`, `villa_operations_log`, `villa_period_documents`, `villa_annual_documents` (`villa_operations.rs:93-162`). Only returns assets the developer is actively linked to (`effective_until IS NULL`).
- Return type: `Json<MatrixDashboardResponse>` with year + assets[] (`villa_operations.rs:60-63`, `212`).
- Cell links generated by JS:
  - Missing: `/developer/villas/${asset_id}/operations/new?year=&month=` (`developer-operations-dashboard.js:154, 385, 590`).
  - Existing log: `/developer/villas/${asset_id}/operations/${log_id}` (`developer-operations-dashboard.js:164, 175, 387, 639`). **Note**: the corresponding edit page route mounted in `mod.rs` is only `/developer/villas/:asset_id/operations/new` (`mod.rs:131-134`) — the `:log_id` URL pattern is **NOT** mounted as a page route. The dashboard links to edit URLs that have no Axum handler.

## 4. Data realism
Fully real. The matrix query joins five live tables (`villa_operations.rs:93-162`); `listed_year/month` derives from `developer_asset_links.effective_from`; `prior_published_count/expected_count` aggregates from prior-year log statuses; `annual_doc_uploaded` checks `villa_annual_documents`. Stats animate from the JSON response in `renderStats()` (`developer-operations-dashboard.js:321-353`). Zero hardcoded sample data.

## 5. Error & empty states
- Initial loading: skeleton (`#ops-skeleton`) with 5 stat shimmers + 4 matrix-row shimmers (`operations-dashboard.html:19-33`).
- Network/HTTP failure: `showError(msg)` hides stats/matrix/empty and reveals `#ops-error` with title `"Failed to load: ${msg}"` (`developer-operations-dashboard.js:731-739`). Error parsing via `responseError(resp)` reads JSON `error`/`message`, falls back to status code (`developer-operations-dashboard.js:741-747`).
- Empty (no villas assigned): `#ops-empty` shown with hero illustration and "View your assets"/"Go to dashboard" CTAs (`operations-dashboard.html:220-346`, `developer-operations-dashboard.js:95-99`).
- Nothing due yet (assigned villas all too recent): info banner with computed "first report covers …" copy (`developer-operations-dashboard.js:230-282`).
- Filter empty: inline "No assets match this filter" row inside the matrix tbody (`developer-operations-dashboard.js:477-489`).
- Banner dismissible per session (`developer-operations-dashboard.js:27-30`, state lost on reload — by design).

## 6. Mobile & responsive

## 8. Functional gaps & dead code
- **Broken edit links**: Matrix cells for `draft`/`submitted`/`rejected`/etc generate hrefs like `/developer/villas/{asset_id}/operations/{log_id}` (`developer-operations-dashboard.js:164, 175, 387, 639`). The route in `mod.rs` (lines 131-134) only mounts `/developer/villas/:asset_id/operations/new` as a page. The log-detail page route is not defined — clicking a non-missing cell likely 404s or hits an unintended handler.
- `currentBillingMonth()` returns the literal current month with no clamp for in-progress periods: developers can submit current-month data, but cells for the current month are tagged "current" while older months use the same overdue logic. Acceptable but worth verifying against the PDF spec referenced in code comments.
- `updateFilterCount('fc-rejected', …)` is not invoked even though the HTML has a "Rejected" tab — only `fc-all`, `fc-action`, `fc-docs` get counts updated (`developer-operations-dashboard.js:446-448`).
- `asset.asset_title.split(',')[1]` is computed but assigned to an empty string on line 499 — dead expression.
- No `TODO`/`FIXME`/`XXX`/`Coming soon`/`Lorem`/`mock`/`fake` markers anywhere.

## 9. Production blockers
**Critical** — Matrix cells for existing logs link to `/developer/villas/:asset_id/operations/:log_id` which has no mounted page route. Either add the route or rewrite the JS to use `…/new?year=&month=` for all cells (effectively the same form, since `operations-submit.js` `hydrate()` queries by `year/month` and pre-populates).
**High** — Zero test coverage (static, API, or E2E) for the entire P2 operations workflow. Critical revenue-reporting surface.
**Medium** — Missing dedicated mobile CSS sheet; relies on inline media queries. "Rejected" filter tab has no count badge wiring.
**Low** — Dead `asset_title.split(',')[1] ? '' : ''` expression. Banner-dismiss state not persisted.

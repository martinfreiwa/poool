★ Recommendation — pick ONE family to unify

my pick: .ds-card as base + .ds-stat-card pattern for KPIs

Across all 14 families, the cleanest cross-platform fit is the .ds-card base + .ds-stat-card stat pattern (already used on developer dashboard, recently applied to operations). It has:

  • One .ds-card base (16px radius, 24px padding, 1px border, subtle shadow)
  • Modifiers for size (--sm), elevation (--elevated), flush (--flush), interactive (--interactive)
  • Composable elements: .ds-card__header, __title, __subtitle, __body, __footer
  • One stat pattern: .ds-stat-card with __label / __value / __trend
  • Glass / holo variant only for premium surfaces

Outside of those: rewards icon-heavy .metric-card stays as opt-in for marketing / referral pages, property cards stay specialized (image-led). Everything else (KPI cards, stat cards, dev-stat, sub-stat, kpi-card, unified-stat-card) can collapse into ds-card + ds-stat-card with body-scoped padding overrides.
